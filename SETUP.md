# Тренировки — установка как PWA через GitHub Pages

Файлы готовы как полноценное приложение (PWA). После установки на iPhone:
- Иконка на главном экране (барбелл на тёмном фоне)
- Открывается в полноэкранном режиме без браузерной строки
- Работает **полностью офлайн** — даже после перезагрузки телефона, даже когда Safari «забудет» обычный кеш

---

## Шаг 1. Структура файлов

Перед заливкой убедись, что у тебя есть такая структура:

```
workout-pwa/
├── index.html
├── manifest.json
├── sw.js
├── icon-180.png
├── icon-192.png
├── icon-512.png
└── gifs/
    ├── goblet-squat.gif
    ├── walking-lunge.gif
    ├── ... (все 20 GIF-ок)
```

GIF-ки у тебя уже есть локально — просто положи их в папку `gifs/` рядом с `index.html`.

---

## Шаг 2. Заливка на GitHub Pages

### Если у тебя нет GitHub

1. Зарегистрируйся на https://github.com (бесплатно)
2. Подтверди email

### Создать репозиторий и залить файлы

**Через веб-интерфейс (без терминала):**

1. На github.com нажми **«+»** в правом верхнем углу → **New repository**
2. Имя репозитория: `workout` (или что хочешь — это будет частью URL)
3. Поставь **Public** (для приватного Pages не работает на бесплатном тарифе)
4. **НЕ** ставь галочку «Add a README»
5. Нажми **Create repository**
6. На странице репозитория нажми **«uploading an existing file»** (синяя ссылка в подсказке)
7. **Перетащи** всю папку `workout-pwa` целиком в окно (или выбери все файлы внутри неё через кнопку «choose your files»)
   - Важно: должны загрузиться и `gifs/`, и все остальные файлы
8. Внизу нажми **Commit changes**

### Включить GitHub Pages

1. В репозитории → вкладка **Settings** (наверху)
2. Слева в меню → **Pages**
3. В разделе **Source** выбери:
   - Branch: **main**
   - Folder: **/ (root)**
4. **Save**

Через 1-2 минуты появится URL вида:
```
https://ТВОЙ-НИК.github.io/workout/
```

Этот URL — твоё «приложение».

---

## Шаг 3. Установка на iPhone

1. Открой Safari (именно Safari, не Chrome) на iPhone
2. Перейди по URL из предыдущего шага
3. Подожди 10-30 секунд — service worker качает все файлы и GIF в постоянное хранилище телефона
4. Тапни кнопку **«Поделиться»** (квадрат со стрелкой вверх внизу)
5. Прокрути → **«На экран „Домой"»**
6. Назови как хочешь («Тренировки») → **Добавить**

Готово. На главном экране иконка с барбеллом. Тапаешь — открывается как настоящее приложение, без браузера. Полностью офлайн.

---

## Как проверить что офлайн работает

После установки на iPhone:
1. Включи режим полёта
2. Открой иконку приложения
3. Перейди по дням → выбери упражнение → должен показаться GIF

Если что-то не показывается — значит этот файл не закешировался. Открой `sw.js` и сверь, есть ли там путь к этому файлу в `PRECACHE_URLS`. Если файла нет в списке — добавь, перезалей на GitHub, в `sw.js` поменяй `CACHE_NAME` с `'workout-app-v1'` на `'workout-app-v2'` (это форсит обновление), снова открой URL на iPhone.

---

## Что если изменишь данные потом

1. Меняешь файлы локально (`index.html`, GIF-ки)
2. Заливаешь обновления на GitHub (через тот же интерфейс upload, или git push)
3. В `sw.js` меняй версию: `'workout-app-v1'` → `'workout-app-v2'`
4. На iPhone открываешь приложение онлайн один раз — service worker сам подхватит обновление

Без смены версии телефон будет показывать старый кеш.

---

## Локальная проверка перед заливкой (опционально)

Хочешь проверить что PWA работает до заливки на GitHub? Запусти локальный HTTP-сервер из папки `workout-pwa`:

**Если есть Python:**
```bash
cd /path/to/workout-pwa
python3 -m http.server 8000
```

Открой в браузере: `http://localhost:8000`

Открой DevTools (Cmd+Opt+I) → вкладка **Application** → **Service Workers** — должно быть зарегистрировано. Там же в **Cache Storage** увидишь все закешированные файлы.

Если работает локально — на GitHub Pages тоже заработает.
